# تجارتي — نظام إدارة المبيعات والمخزون

برنامج سطح مكتب عربي لإدارة المبيعات والمخزون للمحلات والشركات الصغيرة والمتوسطة.
يعمل بدون إنترنت، وبياناته محلية في قاعدة SQLite على الجهاز.

## المميزات

- نقطة بيع (POS): بحث بالاسم أو مسح باركود، سلة، خصومات، وطباعة فورية للفاتورة.
- إدارة المخزون: أصناف، تصنيفات، باركود، أسعار التكلفة والبيع، وتنبيهات نفاد.
- الفواتير: سجل كامل للمبيعات مع إعادة الطباعة والبحث.
- العملاء والموردون: متابعة الديون والمستحقات وتسديدها.
- المشتريات: تسجيل فواتير الشراء (تزيد المخزون تلقائياً).
- لوحة تحكم: إحصائيات لحظية، رسم بياني لمبيعات آخر 14 يوماً، وأفضل المنتجات.
- التقارير: تحليل المبيعات والأرباح حسب الفترة مع تصدير إلى Excel/CSV.
- المستخدمون والصلاحيات: مدير عام / مدير / كاشير.
- تخصيص: اسم الشركة، العملة، الضريبة، وملاحظات الفاتورة.
- واجهة عربية كاملة (RTL) بتصميم داكن.

## التشغيل من المصدر

```bash
pip install -r requirements.txt
python main.py
```

### بيانات الدخول التجريبية

| المستخدم  | كلمة المرور | الصلاحية               |
| --------- | ----------- | ---------------------- |
| `admin`   | `admin`     | مدير عام (كل الصلاحيات) |
| `cashier` | `1234`      | كاشير (بيع فقط)        |

عند أول تشغيل تُنشأ قاعدة بيانات تجريبية فيها منتجات وعملاء وفواتير لعرض البرنامج.
احذف `data/tijarati.db` للبدء بقاعدة فارغة.

## بناء نسخة تنفيذية (.exe)

يُوزَّع البرنامج كملف `.exe` واحد لا يحتاج العميل لتثبيت بايثون.

```bash
pip install pyinstaller
build_exe.bat
```

الملف التنفيذي في مجلد `dist/`.

## بنية المشروع

```
برنامج_المبيعات/
  main.py              نقطة التشغيل
  requirements.txt
  build_exe.bat        سكربت بناء exe
  دليل_المستخدم.md      دليل الاستخدام للعميل
  data/                قاعدة البيانات (تُنشأ تلقائياً)
  app/
    config.py          الإعدادات والألوان ومساعد العربية RTL
    database.py        قاعدة البيانات ومنطق العمل
    widgets.py         عناصر واجهة مشتركة
    login.py           شاشة الدخول
    main_window.py     النافذة الرئيسية والتنقل
    pages/             وحدات البرنامج (POS، فواتير، مخزون، تقارير...)
```

## ملاحظات

- كل البيانات محلية على جهاز العميل، لا تُرسل إلى أي خادم.
- كلمات المرور مخزّنة كـ SHA-256.
- يُنصح بأخذ نسخة احتياطية من `data/tijarati.db` دورياً.
